API

The omnidator provides a simple CORS-enabled API to translate from various formats (currently: microdata, CSV) that use Schema.org terms to RDF as follows:

http://omnidator.appspot.com/translate?url={URL}

For example, invoking the omnidator API as follows (for a CSV file that uses Schema.org terms in the header):

http://omnidator.appspot.com/translate?url=https://raw.github.com/mhausenblas/schema-org-rdf/master/tools/schema-gateway/test/solar-system.csv

What is omnidator about?

Conceptually, omnidator takes whatever kind of data format that uses Schema.org terms and turns it into any other kind of data format. Internally, omnidator uses the Schema.org Gateway, which, in turn, uses Ed Summers' rdflib-microdata plugin for processing microdata. The omnidator eats any kind of data formats - well, actually at the time only microdata and CSV - and translates it to the data lingua franca RDF. It then translates the data to any other target data formats. Yeah, OK, again, currently only native RDF/XML is supported for the output, but we're working on the others, srsly ;).
